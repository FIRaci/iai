## Tổng quan

<Note type="info">
**Playwright** là framework end-to-end testing của Microsoft, hỗ trợ Chromium, Firefox, WebKit. Cho phép viết và chạy automated browser tests với auto-wait, network interception, và multi-browser support.
</Note>

## Cài đặt

### Khởi tạo Playwright project

<CodeBlock language="powershell" code={`npm init playwright@latest`} />

Quá trình setup sẽ hỏi:
- Thư mục chứa tests (mặc định: `tests/`)
- Có thêm GitHub Actions workflow?
- Có cài browser? `npx playwright install`

### Cài đặt thủ công

<CodeBlock language="powershell" code={`npm install --save-dev @playwright/test
npx playwright install`} />

## Sử dụng

### File test cơ bản

<CodeBlock language="typescript" code={`import { test, expect } from '@playwright/test';

test('homepage has title', async ({ page }) => {
  await page.goto('https://example.com');
  await expect(page).toHaveTitle(/Example/);
});

test('click and navigate', async ({ page }) => {
  await page.goto('https://example.com');
  await page.click('a');
  expect(page.url()).toContain('https');
});`} />

### Chạy tests

<CodeBlock language="powershell" code={`# Chạy tất cả tests
npx playwright test

# Chạy với UI mode
npx playwright test --ui

# Chạy với specific browser
npx playwright test --project=chromium

# Chạy và xem report
npx playwright show-report`} />

### Cấu hình playwright.config.ts

<CodeBlock language="typescript" code={`import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,
  retries: 2,
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  projects: [
    { name: 'chromium', use: { browserName: 'chromium' } },
    { name: 'firefox', use: { browserName: 'firefox' } },
    { name: 'webkit', use: { browserName: 'webkit' } },
  ],
});`} />

## Lưu ý

<Note type="warning">
Playwright tự động download browser binaries (~200-400MB mỗi browser). Nếu muốn dùng browser đã cài sẵn, dùng `PLAYWRIGHT_BROWSERS_PATH=0`.
</Note>

<Note type="tip">
Dùng `--ui` mode để xem test chạy real-time, inspect DOM state sau mỗi step, và debug dễ dàng hơn.
</Note>
