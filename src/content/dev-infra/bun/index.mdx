## Tổng quan

Bun là all-in-one JavaScript runtime, bundler, transpiler, và package manager được viết bằng Zig dựa trên JavaScriptCore engine. Có tốc độ chạy script nhanh hơn Node.js từ 2-4x, tích hợp sẵn built-in bundler, SQLite client, và test runner.

## Cài đặt

### Phương pháp 1: Script install (khuyến nghị)

<CodeBlock language="powershell" code={`powershell -c "irm bun.sh/install.ps1 | iex"`} />

### Phương pháp 2: npm

<CodeBlock language="powershell" code={`npm install -g bun`} />

### Phương pháp 3: Chocolatey

<CodeBlock language="powershell" code={`choco install bun`} />

## Sử dụng

### Chạy JavaScript/TypeScript file

<CodeBlock language="powershell" code={`bun run app.ts       # Chạy trực tiếp TypeScript
bun run index.js      # Chạy JavaScript
bun --watch server.ts # Chạy với hot-reload`} />

### Package management

<CodeBlock language="powershell" code={`# Install dependencies (tương thích package.json)
bun install

# Thêm package
bun add express
bun add -d typescript

# Gỡ package
bun remove lodash

# Chạy script
bun run dev`} />

### Test runner

<CodeBlock language="powershell" code={`// math.test.ts
import { describe, test, expect } from "bun:test";

describe("math", () => {
  test("add", () => {
    expect(1 + 1).toBe(2);
  });
});

# Chạy test
bun test`} />

### HTTP Server

<CodeBlock language="typescript" code={`// server.ts
Bun.serve({
  port: 3000,
  fetch(req) {
    return new Response("Hello from Bun!");
  },
});`} />

## Lưu ý

<Note type="tip">
Bun có built-in SQLite client (`bun:sqlite`), built-in test runner (`bun test`), và built-in bundler — giảm dependency bên ngoài đáng kể.
</Note>

<Note type="warning">
Một số packages Node.js native (như `sharp`, `node-canvas`) chưa tương thích hoàn toàn với Bun. Kiểm tra compatibility tại [bun.sh/docs/runtime/nodejs-apis](https://bun.sh/docs/runtime/nodejs-apis) trước khi chuyển đổi.
</Note>
