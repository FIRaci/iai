## Tổng quan

npm (Node Package Manager) là package manager mặc định của Node.js, dùng để cài đặt, chia sẻ và quản lý các JavaScript packages từ npm registry. Hiện là registry lớn nhất thế giới với hơn 2 triệu packages.

## Cài đặt

npm đi kèm với Node.js. Nếu đã cài Node.js, npm đã có sẵn.

### Kiểm tra npm

<CodeBlock language="powershell" code={`npm --version
# Kết quả mẫu: 10.x.x`} />

### Cập nhật npm lên phiên bản mới nhất

<CodeBlock language="powershell" code={`npm install -g npm@latest`} />

## Sử dụng

### Cài đặt packages

<CodeBlock language="powershell" code={`# Cài đặt global
npm install -g typescript

# Cài đặt local (production dependency)
npm install express

# Cài đặt local (dev dependency)
npm install --save-dev typescript

# Cài đặt phiên bản cụ thể
npm install react@18.3.1`} />

### Quản lý dependencies

<CodeBlock language="powershell" code={`# Khởi tạo project mới
npm init -y

# Cài tất cả dependencies từ package.json
npm install

# Gỡ package
npm uninstall lodash

# Cập nhật packages
npm update

# Kiểm tra outdated packages
npm outdated`} />

### Chạy scripts

<CodeBlock language="powershell" code={`# package.json scripts
npm run dev
npm run build
npm run test

# Chạy trực tiếp package không cần cài (npx)
npx create-react-app my-app
npx tsc --init`} />

## Xác minh cài đặt

<CodeBlock language="powershell" code={`npm --version
# Kiểm tra packages đã cài global
npm list -g --depth=0`} />

## Lưu ý

<Note type="tip">
Dùng `npx` thay vì `npm install -g` để chạy package mà không làm đầy global namespace. Ví dụ: `npx create-react-app my-app`.
</Note>

<Note type="warning">
Tránh dùng `npm install -g` quá nhiều. Ưu tiên cài local cho từng project để tránh xung đột phiên bản.
</Note>
