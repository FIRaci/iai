## Tổng quan

pnpm là package manager cho JavaScript/Node.js nhanh hơn và tiết kiệm disk space hơn npm. Sử dụng cơ chế content-addressable storage với hard links để chia sẻ packages giữa các projects, giúp tiết kiệm đến 70% dung lượng ổ đĩa.

## Cài đặt

### Phương pháp 1: Winget

<CodeBlock language="powershell" code={`winget install pnpm.pnpm`} />

### Phương pháp 2: npm global

<CodeBlock language="powershell" code={`npm install -g pnpm`} />

### Phương pháp 3: Corepack (Node.js 16.13+)

<CodeBlock language="powershell" code={`corepack enable
corepack prepare pnpm@latest --activate`} />

## Sử dụng

### Lệnh cơ bản

<CodeBlock language="powershell" code={`# Khởi tạo project
pnpm init

# Cài dependencies
pnpm install

# Thêm package
pnpm add express

# Thêm dev dependency
pnpm add -D typescript

# Gỡ package
pnpm remove lodash`} />

### Workspace (Monorepo)

<CodeBlock language="powershell" code={`# Tạo pnpm-workspace.yaml
packages:
  - "packages/*"
  - "apps/*"

# Cài tất cả dependencies cho workspace
pnpm install

# Chạy script trong workspace package
pnpm --filter @myapp/server run dev`} />

### So sánh lệnh với npm

| npm | pnpm |
|-----|------|
| `npm install` | `pnpm install` |
| `npm install [pkg]` | `pnpm add [pkg]` |
| `npm install -D [pkg]` | `pnpm add -D [pkg]` |
| `npm uninstall [pkg]` | `pnpm remove [pkg]` |
| `npm run [script]` | `pnpm [script]` |

## Xác minh cài đặt

<CodeBlock language="powershell" code={`pnpm --version
# Kết quả mẫu: 10.x.x`} />

## Lưu ý

<Note type="tip">
pnpm tạo `node_modules` dạng symlink — thay vì copy, nó link đến store global. Điều này giúp mỗi project chỉ lưu 1 bản vật lý của mỗi package version.
</Note>

<Note type="warning">
Khi chuyển từ npm sang pnpm, hãy xóa `node_modules` và `package-lock.json` trước khi chạy `pnpm install` lần đầu.
</Note>
