## Tổng quan

RAGFlow là open-source RAG engine được thiết kế cho document parsing chất lượng cao. Khác với các RAG tool khác, RAGFlow parse sâu cấu trúc tài liệu (bảng biểu, header, hình ảnh trong PDF) thay vì chỉ extract text thô. Hỗ trợ template-based retrieval cho phép tùy chỉnh cách trích xuất thông tin theo từng loại tài liệu.

## Sử dụng

1. **Upload documents:** Hỗ trợ PDF, DOCX, Excel, TXT, HTML, Markdown, hình ảnh. Vào Knowledge Base -> Create Dataset -> Upload files.
2. **Document parsing:** RAGFlow tự động parse bảng biểu, hình ảnh, công thức toán học. Chọn parsing method: AQA (trả lời trực tiếp), Knowledge Graph (trích entity-relation), hoặc General.
3. **Chunking configuration:** Cấu hình chunk size (mặc định 512), overlap (mặc định 64), chunk method: dấu câu, section, hoặc custom delimiter.
4. **Embedding model:** Chọn embedding model trong Settings -> Model Providers. Hỗ trợ OpenAI, BAAI/bge, text2vec-large-chinese, và nhiều model local qua Ollama.
5. **Query:** Tạo assistant và chat qua UI hoặc API.

<CodeBlock language="powershell" code={`
# API query
curl -X POST http://localhost:9380/api/v1/retrieval -H 'Content-Type: application/json' -d '{"question":"Cach cai dat RAGFlow tren Windows?","dataset_ids":["ds-xxx"],"top_k":5}'
`} />

## Lưu ý

RAGFlow chạy tốt nhất với document tiếng Trung và tiếng Việt nhờ model parsing chuyên biệt. Với PDF có bảng biểu phức tạp, nên dùng parsing mode AQA. Nếu chunk size quá nhỏ, thông tin bị thiếu context; nếu quá lớn, retrieval kém chính xác. Có thể tích hợp Ollama để chạy embedding model local không cần GPU.
